## 1. 引言

Spanwork（以下简称「本应用」或「我们」）是一款**本地优先**的项目管理与时间记录应用，帮助您管理目标式项目、习惯式项目、任务与时间记录，并可在**同一局域网内**与您的其他设备进行可选的数据同步。

我们重视您的隐私。本政策说明本应用如何处理与您相关的信息。请在使用本应用前仔细阅读。

**重要提示**：本应用默认**不注册账号、不连接我们运营的云服务器、不上传您的业务数据**。您的项目、任务、习惯与时间记录主要保存在您设备本地的数据库中，由您自行掌控。

---

## 2. 适用范围

本政策适用于：

- Spanwork macOS 桌面应用（Bundle ID：`com.spanwork.desktop`）
- Spanwork Windows 桌面应用
- Spanwork iOS 应用（Bundle ID：`com.spanwork.app`）
- Spanwork Android 应用

本政策不适用于第三方网站或服务（例如 Apple App Store、TestFlight）的隐私实践；使用这些服务时，请同时查阅相应平台的隐私政策。

## 3. 我们收集与存储的信息

### 3.1 您主动创建的内容（本地存储）

在您使用本应用的过程中，以下内容会写入设备本地的 SQLite 数据库（`spanwork.db`），**默认不会发送给我们或任何第三方服务器**：

| 类别 | 示例内容 |
| --- | --- |
| 项目与分类 | 项目名称、描述、类型（目标式/习惯式）、颜色、图标、日期区间、归档状态 |
| 任务与里程碑 | 任务标题、描述、状态、优先级、截止日期、标签、层级关系 |
| 习惯 | 习惯规则、打卡实例、连续记录、行为设计相关字段 |
| 时间记录 | 计时起止时间、时长、关联任务/习惯、备注、来源（计时器/手动补录） |
| 设备标识（本地） | 本机生成的设备 ID、您可修改的设备名称、平台类型（如 macOS/iOS/Windows） |
| 同步相关（本地） | 已配对设备信息、同步游标、同步会话日志（用于排查同步问题） |

上述数据属于**您个人创建或使用本应用产生的信息**，其内容取决于您如何填写和使用本应用。

### 3.2 我们不收集的信息

在正常使用本应用时，我们**不会**主动收集以下信息：

- 姓名、电子邮箱、电话号码等注册身份信息（本应用无需注册账号）
- 精确地理位置（GPS）
- 通讯录、相册、麦克风、相机内容
- 广告标识符或跨应用追踪数据
- 用于用户画像的第三方分析/统计信息

本应用**不包含**第三方广告 SDK、Crash 上报 SDK 或用户行为分析 SDK。

### 3.3 开发与诊断信息（仅开发构建）

在**开发版（Debug）**构建中，应用界面可能显示版本号、运行日志等诊断入口，便于开发者排查问题。  
在**正式发布（Release）**构建中，上述诊断入口默认不向普通用户展示。  
即使存在本地日志，其内容也**不会自动上传**至我们的服务器。

## 4. 信息的使用方式

我们仅在实现产品功能所必需的范围内使用本地存储的信息，例如：

- 展示与管理您的项目、任务、习惯与时间记录
- 计算今日进度、累计时间、日历视图等统计展示
- 在您发起同步时，与已配对的设备交换并合并数据
- 在本地记录同步历史，便于您了解同步结果

我们**不会**将您的业务数据用于广告投放、出售给第三方，或用于与本应用核心功能无关的目的。

## 5. 局域网同步

本应用提供**可选的**局域网（LAN）同步功能。仅在您主动开启并使用同步相关功能时，才会涉及设备间通信。

### 5.1 同步过程中可能交换的信息

- 设备 ID、设备名称、平台类型、应用/协议版本（用于识别对端与握手）
- 您在应用中创建并已纳入同步范围的项目、任务、习惯、时间记录等业务数据
- 为合并冲突所需的字段级变更元数据（如更新时间、来源设备 ID）

### 5.2 发现与连接方式

- **mDNS/Bonjour**：在局域网内广播与发现 `_spanwork._tcp` 服务（iOS 需「本地网络」权限）
- **手动连接**：您输入 IP 地址与端口主动连接
- **热点探测**：在特定网络环境（如 iPhone 个人热点网段）下尝试发现对端设备
- **配对码**：配对时需双方确认，以降低误连风险

### 5.3 重要说明

- 同步发生在**您与您的设备之间**，或**同一局域网内的设备之间**，不经过 Spanwork 运营的云端中转服务器。
- 同步内容可能包含您输入的文本（如项目名称、任务描述、备注）。请确保您有权在设备间共享这些信息。
- 公共 Wi‑Fi 等不可信网络环境下，同步数据可能被同网段其他方嗅探。建议在可信网络中使用同步，或避免在公共网络开启同步。

## 6. 系统权限说明

### iOS

| 权限 | 用途 |
| --- | --- |
| 本地网络（Local Network） | 在局域网内发现 Spanwork 对端设备并完成数据同步 |

### macOS / Windows

桌面端同步同样使用本地网络能力；具体系统提示因平台而异。除实现同步与本地数据存储外，本应用不要求额外敏感权限。

## 7. 数据存储位置与保留

- **存储位置**：数据保存在您设备上由操作系统分配的应用数据目录中（例如 iOS 应用沙盒、macOS/Windows 的 App Data 目录）。
- **保留期限**：数据在您未删除前会一直保留在本地；卸载本应用通常会导致本地数据被系统清除（具体行为取决于操作系统）。
- **备份**：若您启用了 iCloud 设备备份、Time Machine 等系统级备份，本地数据库可能随系统备份一并保存，该部分由您与平台提供商自行管理。

---

## 8. 数据的删除与导出

- **删除内容**：您可在应用内删除项目、任务、习惯、时间记录等；删除操作作用于本地数据库（同步后可能影响已配对设备上的副本，取决于同步合并结果）。
- **删除全部数据**：卸载应用，或在系统设置中清除应用数据（若平台支持），可移除本地存储的数据。
- **导出**：当前版本以本地使用为主，未提供统一的「一键导出全部数据」入口。若后续版本增加导出功能，将在此政策或产品说明中更新。

## 9. 第三方服务

本应用核心功能**不依赖**第三方云端 API 处理您的业务数据。

在以下场景中，您可能间接使用第三方服务，其隐私实践由相应平台负责：

| 场景 | 说明 |
| --- | --- |
| Apple App Store / TestFlight | 通过 Apple 分发或测试 iOS 版本时，Apple 可能收集与账户、下载、崩溃报告相关的信息 |
| GitHub Releases | 若您从 GitHub 下载桌面安装包，GitHub 可能记录访问与下载日志 |
| 操作系统服务 | 系统更新、安全机制、备份服务等由 Apple / Microsoft 等平台提供 |

除上述分发与系统层面外，我们不会将您的项目数据委托给第三方处理。

## 10. 信息安全

我们采取合理措施保护本地数据，例如：

- 使用嵌入式 SQLite 数据库，数据存于应用沙盒/用户数据目录
- 同步连接在用户配对确认后进行，减少未授权设备接入
- 局域网通信限于实现同步协议所需的最小信息交换

但请注意：**没有任何传输或存储方式能保证绝对安全**。您应妥善保管设备、操作系统账户，并在共享设备上谨慎使用本应用。

## 12. 跨境传输

本应用默认将数据保存在**您设备所在地区**。  
仅在您主动使用局域网同步时，数据会在您控制的设备之间传输；**不会**因使用本应用核心功能而默认传输至境外服务器。
